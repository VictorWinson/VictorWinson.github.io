learn to use git

git

[toc]

Jargon#

  • Tree
    folder

  • Blob
    file

  • Working tree
    Where all files exist

dose it contains files that is not added?

  • Staging area(Index)
    What added and are going to be contain in next commit will be in the staging area

  • Repository
    Database that stores the history of the project

basic use#

Tree status#

  1. Dirty/modified
    None of the modified files have been added for next commit

  2. Staged
    Some of the files have been added and can be commit now

  3. Committed
    Change have been stored safely in a snapshot(commit)

File status#

  1. Tracked
    Were part of last snapshot or newly added to the stage

    • modified: have changed but not yet added
    • unmodified: haven’t changed
    • staged: have changed and add to stage or newly added to the stage

  2. Untracked
    Not part of last snapshot nor are in the staging area

commands#

git commit#

git commit -a

include all the tracked files including modified but not staged ones

git commit -m

add a commit message in the commit command rather than use a default editor to write one

git stash#

To store dirty files in a special stack

git stash

to store all dirty files

git stash list

show previous stashes

git stash apply

apply latest stash

git stash apply stash@{num}

apply specific stash

git stash drop

same as apply only clear the stash instead of applying

git stash pop

same as apply only drop it after apply it

git stash clear

clear all the stashes

git merge#

Merge two branches

git fetch#

To get the remote branches that not on local repository

git pull#

Combination of git fetch and git merge

git push#

Push a branch to remote

git push <remote> <local branch>:<rename as>

push a brach to remote but use a different name

git branch#

git branch -u <remote>/<remote branch>

set up to stream with remote branch

git rebase#

Redo the changes made to one branch since the branching to the another branch. This will make the changes are made in series not in parallel

git remote#

git remote add origin [git@url:port]:path/project.git

# change the remote HEAD of origin to develop branch 
git remote set-head origin develop

# set default push branch
git branch --set-upstream-to=origin/mac

git checkout#

git checkout change into the situation of a commit
-b create a new branch and change into it

git diff#

git diff diff a file between different commit

Branching change and merging#

If meeting merge conflict, try to automatically fix it, if can’t, report it.
