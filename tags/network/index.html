net mask#

  • which ranger of ip are local that don’t need to go through a router
  • when you send a package and whenever the netmask is 1 the corresponding binary digits of you ip address and desitination address are the same you are in the same local network.

address structure#

  • Class A: first bit is 0 and next 7 bit is network and last 24 is hosts
  • Class B: first 2 bit is 1 0 and next 14 bit is network and last 16 is hosts
  • Class C: first 3 bit is 1 1 0 and next 21 bit is network and last 8 is hosts

Classless Inter-Domain Routing(CIDR)#

  • use slash and a number to descibe the length of the prefix(netmask)
  • /16 means the netmask is 16 bit of binary number which left 16 bit which is 65536 ipv4 addresses under the domain.
  • similarly, /24 describes 256 addresses and /20 describes 4096 address.

longest prefix match(algorithm)#

  • it is a algorithm to decide how to forward a packet
  • if a packet arrives and there isn’t a more specific match in the forwarding table than the default one, it will sent it through the default one.
  • forwarding table is a set of CIDR entries. an address can match multiple entries. base on prefix length to decide which is more specific.

ARP (Address Resolution Protocol)#

  • only works in local network
  • every node keep a cache of mapping of IP addresses to the link layer addresses(MAC).
  1. When the node need to send a packet to an IP address:
    • If it has it in the cache it send it directly.(jump to 4)
    • If it does not have the cache it will broadcast a request that ask “who have IP x.x.x.x tell x.x.x.x(itself) at xx:xx:xx:xx:xx:xx(MAC)”
  2. The node that have the specific IP will respond “I hve network address x.x.x.x and my link layer addresses is xx:xx:xx:xx:xx:xx”(not broadcast). And to the rest nodes in the network when they overhear the request, they can insert or refresh their ARP mapping in the cache.
  3. The node that send the request will generate a mapping of link layer addresses and IP addresses.

3 hand shakes#

  1. SYN (synchronize)
    • Send along with a number that identified where to start the byte-stream.
  2. SYN+ACK (acknowledge)
    • Acknowledge of agreeing to communicate with it. And also send a SYN back for reverse direction connection.
  3. ACK
    • Acknowledge of agreeing to the reverse direction connection.
